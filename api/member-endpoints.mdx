Member endpoints power the dashboard and mobile app experience. They cover the full lifecycle of a member's relationship with The Telescope Net: creating an account, linking telescope nodes, generating activation codes for first-boot registration, browsing observation history and cumulative statistics, managing node schedules and vacation modes, and controlling notification preferences. All endpoints except registration and login require an `Authorization: Bearer <token>` header obtained at login.

---

## POST /api/v1/auth/register

Create a new member account. Returns a bearer token immediately so the client does not need a separate login step.

<Tabs>
  <Tab title="Request">
    **Auth:** None

    <ParamField body="email" type="string" required>
      Valid email address. Must contain `@` and a dot in the domain. Stored in lowercase. Returns `400` if the address is already registered.
    </ParamField>

    <ParamField body="password" type="string" required>
      Minimum 8 characters. Hashed server-side with PBKDF2-HMAC-SHA256 at 260,000 iterations with a per-user random salt.
    </ParamField>

    <ParamField body="display_name" type="string">
      Optional display name shown in the member dashboard. Defaults to the email prefix if omitted.
    </ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="user_id" type="string">Unique member identifier, e.g. `"u_4a8b1c2d"`.</ResponseField>
    <ResponseField name="token" type="string">Bearer token. Pass in `Authorization: Bearer <token>` on all subsequent member requests.</ResponseField>
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/auth/register \
         -H "Content-Type: application/json" \
         -d '{"email": "you@example.com", "password": "SecurePass123!", "display_name": "Jane"}'
    ```

    ```json
    {
      "user_id": "u_4a8b1c2d",
      "token": "aBcDeFgHiJkLmNoPqRsTuVwXyZ012345_abc"
    }
    ```
  </Tab>
</Tabs>

## POST /api/v1/auth/login

Authenticate with email and password. Issues a **new** bearer token, invalidating any previously issued token for this account.

<Tabs>
  <Tab title="Request">
    **Auth:** None

    <ParamField body="email" type="string" required>Registered email address.</ParamField>
    <ParamField body="password" type="string" required>Account password.</ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="user_id" type="string">Member identifier.</ResponseField>
    <ResponseField name="token" type="string">Fresh bearer token.</ResponseField>

    **401** is returned with a deliberately vague `"invalid email or password"` error for both unknown emails and wrong passwords, to prevent account enumeration.
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/auth/login \
         -H "Content-Type: application/json" \
         -d '{"email": "you@example.com", "password": "SecurePass123!"}'
    ```

    ```json
    {
      "user_id": "u_4a8b1c2d",
      "token": "new_token_here"
    }
    ```
  </Tab>
</Tabs>

## DELETE /api/v1/me

Permanently delete the member account and all associated data including notifications, activation codes, node memberships, and help chat history. This action is irreversible.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField body="confirm" type="boolean" required>
      Must be `true`. The server returns `400` if this field is absent or false, preventing accidental deletion.
    </ParamField>
  </Tab>

  <Tab title="Example">
    ```bash

         -H "Content-Type: application/json" \

         https://api.thetelescope.net/api/v1/me
    ```

## POST /api/v1/me/nodes/{node_id}

Claim an existing node by presenting its `api_key`. Use this to link a node that was registered before the member account existed, or to add a second account as a co-owner.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField path="node_id" type="string" required>The node identifier to claim.</ParamField>
    <ParamField body="api_key" type="string" required>The node's secret API key (from `data/cloud_state.json` on the node).</ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="ok" type="boolean">`true` when the claim is recorded.</ResponseField>
    <ResponseField name="node_id" type="string">The claimed node ID.</ResponseField>

    **401** is returned when the `api_key` does not match the node.
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/me/nodes/node_a3f9b2c1 \
         -H "Authorization: Bearer <token>" \
         -H "Content-Type: application/json" \
         -d '{"api_key": "<the_node_api_key>"}'
    ```

    ```json
    {"ok": true, "node_id": "node_a3f9b2c1"}
    ```
  </Tab>
</Tabs>

## POST /api/v1/me/nodes/{node_id}/session

Start a portable node's observing session for tonight. Updates the session location, fetches sky quality for that location, and sets the node status to `active`. Only valid for nodes registered as `portable`.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField path="node_id" type="string" required>Node identifier. Must be a portable node claimed by this member.</ParamField>

    <ParamField body="lat" type="float" required>Tonight's observing latitude in decimal degrees.</ParamField>
    <ParamField body="lon" type="float" required>Tonight's observing longitude in decimal degrees.</ParamField>

    <ParamField body="city" type="string">
      Human-readable city or locality name for this session location.
    </ParamField>

    <ParamField body="site_name" type="string">
      Optional site name, e.g. `"Cherry Springs State Park"`.
    </ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="ok" type="boolean">`true` when the session is started.</ResponseField>
    <ResponseField name="mpsas" type="float">Sky brightness at tonight's location in magnitudes per square arcsecond.</ResponseField>
    <ResponseField name="bortle" type="integer">Bortle scale classification for the session site (1–9).</ResponseField>

    **400** is returned when `lat` or `lon` are missing, or for non-portable nodes.

    **404** is returned when the node is not claimed by this member.
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/me/nodes/node_a3f9b2c1/session \
         -H "Authorization: Bearer <token>" \
         -H "Content-Type: application/json" \
         -d '{"lat": 41.6611, "lon": -77.8202, "city": "Coudersport", "site_name": "Cherry Springs SP"}'
    ```

    ```json
    {"ok": true, "mpsas": 21.8, "bortle": 2}
    ```
  </Tab>
</Tabs>

## PUT /api/v1/me/nodes/{node_id}/vacation

Pause a node's observation schedule until a specified date. Nights during the vacation window are excluded from the reliability score calculation. The scheduler will not generate plans for a node on vacation.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField path="node_id" type="string" required>Node identifier. The node must be claimed by this member.</ParamField>

    <ParamField body="until_date" type="string" required>
      ISO date string in `YYYY-MM-DD` format specifying the last day of vacation.
    </ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="ok" type="boolean">`true` when vacation is set.</ResponseField>
    <ResponseField name="vacation_until" type="string">The date that was set.</ResponseField>

    **400** is returned when `until_date` is missing or not in `YYYY-MM-DD` format.

    **404** is returned when the node is not claimed by this member.
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X PUT \
         -H "Authorization: Bearer <token>" \
         -H "Content-Type: application/json" \
         -d '{"until_date": "2026-07-01"}' \
         https://api.thetelescope.net/api/v1/me/nodes/node_a3f9b2c1/vacation
    ```

    ```json
    {"ok": true, "vacation_until": "2026-07-01"}
    ```
  </Tab>
</Tabs>

## POST /api/v1/me/activation-code

Generate a personal activation code for linking a new node to this account. The code is embedded in the installer flow and is consumed during `POST /api/v1/nodes/register`. Codes expire after **30 days**.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField body="location_name" type="string">
      Place name for the observatory, e.g. `"Rockwood TX"`. Geocoded via Nominatim and stored with the code so the node automatically receives coordinates at registration.
    </ParamField>

    <ParamField body="latitude" type="float">
      Observatory latitude. Takes precedence over `location_name` geocoding when provided together with `longitude`.
    </ParamField>

    <ParamField body="longitude" type="float">
      Observatory longitude.
    </ParamField>

    <ParamField body="telescope_model" type="string">
      Telescope model chosen in the app, e.g. `"ZWO Seestar S50"`. Used to seed hardware spec defaults at registration.
    </ParamField>

    <ParamField body="telescope_display_name" type="string">
      Human-readable name for this specific telescope, max 80 characters.
    </ParamField>

    <ParamField body="portable" type="boolean">
      `true` if this will be a portable node.
    </ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK**

    <ResponseField name="code" type="string">Activation code in `BS-YYYY-XXXXXXXX` format.</ResponseField>
    <ResponseField name="expires_at" type="string">ISO 8601 UTC expiry timestamp (30 days from generation).</ResponseField>
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/me/activation-code \
         -H "Authorization: Bearer <token>" \
         -H "Content-Type: application/json" \
         -d '{
           "location_name": "Rockwood TX",
           "telescope_model": "ZWO Seestar S50",
           "telescope_display_name": "Back Porch Seestar"
         }'
    ```

    ```json
    {
      "code": "BS-2026-ABCD1234",
      "expires_at": "2026-07-15T14:22:00.000000+00:00"
    }
    ```
  </Tab>
</Tabs>

## GET /api/v1/me/observations

Retrieve photometry measurements uploaded by all nodes owned by this member, most recent first.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField query="days" type="integer">
      Look-back window in days. Default `90`, maximum `365`.
    </ParamField>

    <ParamField query="limit" type="integer">
      Maximum number of rows to return. Default `200`, maximum `100,000`.
    </ParamField>
  </Tab>
  <Tab title="Response">

## GET /api/v1/me/timeline

        <ResponseField name="target" type="string">Target name.</ResponseField>

## POST /api/v1/me/help/chat

Send a message to the OpenRouter-powered help assistant. Members are limited to 5 user messages per week. The assistant has access to the member's node configuration context and can generate configuration patches that the node agent applies automatically.

<Tabs>
  <Tab title="Request">
    **Auth:** `Authorization: Bearer <token>`

    <ParamField body="message" type="string" required>
      The help message text to send to the assistant.
    </ParamField>

    <ParamField body="node_id" type="string">
      Optional node ID to scope the assistant's context to a specific node's configuration.
    </ParamField>
  </Tab>
  <Tab title="Response">
    **200 OK** — Returns the assistant's reply and any generated config patches.

    **400** is returned when the message is empty.

    **429** is returned when the weekly quota is exhausted.

    **503** is returned when the upstream AI provider is unavailable.
  </Tab>
  <Tab title="Example">
    ```bash
    curl -X POST https://api.thetelescope.net/api/v1/me/help/chat \
         -H "Authorization: Bearer <token>" \
         -H "Content-Type: application/json" \
         -d '{"message": "My node keeps failing to slew past azimuth 180. How do I set a horizon mask?", "node_id": "node_a3f9b2c1"}'
    ```
  </Tab>
</Tabs>
